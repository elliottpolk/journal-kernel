# Decision-Making Principles

## Eliminate "Should": Force Accountability and Prioritization

**"Should" is accountability avoidance.** It is passive, vague, and lets decisions evaporate without ownership or priority. When you catch yourself or others using "should," stop and force clarity by acknowledging the nuance of both **who owns the decision** and **how it's prioritized**:

- **NEED/MUST**: Non-negotiable requirement or constraint with clear ownership
  - Example: "I MUST finish this before the deadline" (not "I should probably finish this")
  - Signals: Highest priority, required, there are consequences for not doing it
  - Prioritization: This wins in trade-off decisions

- **WANT/PREFER**: Desired outcome with explicit prioritization and decision-maker
  - Example: "I WANT to learn Rust for this project" (not "I should probably learn Rust")
  - Signals: Medium priority, preferred path, alternatives exist, you own choosing
  - Prioritization: This can be traded against other WANTs or deferred if NEEDs conflict

- **MAY/CAN**: Option that requires an active decision from the receiving party
  - Example: "You MAY defer this to next week if other priorities conflict" (not "you should probably defer this")
  - Signals: Lower priority, decision is in your court, you own the choice
  - Prioritization: This explicitly acknowledges trade-offs and puts decision power with the
    receiver

**Why this matters:**
- "Should" fails on both dimensions: no clear owner AND no clear priority
- This framework provides both simultaneously: accountability (who owns it) + prioritization (how important is it)
- When resources are constrained or conflicts arise, the combination enables clear action
- "MAY/CAN" forces an explicit choice (yes/no, do/don't) with ownership

## Applying the Framework

When making or recording a decision:

1. Identify the level: NEED, WANT, or MAY
2. Name the owner: who holds this requirement or choice
3. State the consequence or trade-off: what happens if this is deferred or dropped
4. Avoid "should" in all agent outputs, memory entries, and workflow instructions
